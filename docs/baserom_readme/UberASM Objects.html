UberASM objects

The baserom uses a framework of custom objects to allow you to toggle or activate some simple UberASM codes that are in the baserom, most of which are very basic utility features. They are activated simply by inserting a custom extension object into whichever levels you want with no real limit. These objects work in conjunction with any level UberASM you may want to use as well.

Available Objects

The following objects can be used to enable or toggle features or settings in the baserom.

#ObjectDescription
98Free Vertical ScrollMakes the camera follow Mario up and down.
99Lock Horizontal ScrollPrevents the camera from scrolling left or right.
9AStart in OFF StateStarts the level with the ON/OFF state in the OFF state on level load.
9BToggle Block DuplicationBlock duplication is patched in the baserom. This object will set a flag to disable the patch that prevents block duplication.
9CToggle StatusbarCompletely hide the in game statusbar, note: this does not disable the item box.
9DToggle Screen ScrollScrolling the screen with L or R is disabled by default in the baserom but this will enable it.
9EEnable SFX EchoEnabled the echo for sound effects, provided your level has custom music with echo enabled.
A0No Power UpsUse this object to prevent power-up collection.
A2Enable Vanilla Cape TurnaroundThe baserom has patched turning around in the air with spin during cape to make it guaranteed on button press. Using this object will disable that consistency.
A3Enable 8-frame Cape FloatCape floating is vanilla in the baserom but using this object will apply a consistent 8 frame counter after the input is registered to adjust Mario's falling speed with cape.
A4Zero Cape Float DelayUsing this object will remove the delay before Mario starts floating with cape.
A5Death on Power-up LossMakes it so losing a power up will kill Mario instantly.
A7Press L & R To DieKills the player if L and R are pressed. Useful for a quick reset.
A9Disable Spin Jump FireballsUse this object to disable Mario's fireballs when spin jumping.

Retry System Settings

Inserting these objects into your level to override the default settings of the retry system or even change the retry settings between sublevels.

#ObjectDescription
B0Instant RetryMake the retry skip using prompt and restart instantly after dying.
B1Prompt RetrySet the retry system to use a prompt that says "Retry" in the middle of the screen.
B2Bottom Left Retry PromptMove the retry prompt to the bottom left (by default it's in the middle) if Prompt retry is active.
B3No Power Ups from MidwaysDisable gaining a power-up (mushroom) from a midway point.
B3Vanilla RetrySet the retry system back to using Vanilla death routine.

Inserting Custom Objects

To insert a custom object either use the "Edit > Insert manual..." menu action or press the Insert key while in Lunar Magic's Object editing mode to bring up the following window.

To insert a custom UberASM object put its value from the tables below in the Size/Type/Ext field seen above. Multiple custom UberASM objects can be active in one level when they are inserted one at time.

When inserted, custom objects will look garbled in Lunar Magic since they load a "null" object but they won't actually appear in game. The tiles they are made up of may occasionally use an exit-enabled (door) tile which may display a warning depending on your Lunar Magic settings.
